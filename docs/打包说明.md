# iOS 打包说明

## 当前限制

当前仓库不能执行源码构建，因为缺少完整 Xcode 工程和源代码。`scripts/package_ios.sh` 中引用了 `amz_profit_calculator.xcodeproj` 和 `amz_profit_calculator` scheme，但仓库里只有共享 scheme 文件，没有 `project.pbxproj`，Xcode 无法识别为可构建工程。

现有 `versions/` 目录下的 IPA 是历史构建产物，可以发布和安装，但不能代表当前仓库可重新编译。

## 发布现有 IPA

`.github/workflows/publish-latest-ipa.yml` 会在推送到 `main` 或手动运行时：

1. 从 `versions/` 目录寻找最新的 `qlmb-adhoc.ipa`。
2. 复制为 Release 资产。
3. 重新创建 `unsigned-latest` Release。
4. 上传真实 `.ipa` 文件，避免下载 GitHub artifact zip 后改后缀导致安装失败。

## 真正构建 IPA 需要补齐

要恢复源码构建，需要至少补齐：

- `amz_profit_calculator.xcodeproj/project.pbxproj` 或真实项目名对应的完整 Xcode 工程
- Swift / Objective-C 源码
- `Info.plist`
- 资源文件、Core Data model、entitlements 等工程依赖
- 可在 `xcodebuild -list` 中识别的 shared scheme

补齐后再运行：

```bash
xcodebuild -list -project amz_profit_calculator.xcodeproj
```

确认工程可识别后，才能继续配置 Archive 和 IPA 导出流程。
