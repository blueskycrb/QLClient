# iOS 打包说明

项目已提供 `scripts/package_ios.sh`，用于本地模拟器构建、Archive、IPA 导出，并按版本生成 `versions/v版本_时间/RELEASE_NOTES.md`，方便保留每次打包记录。

## 前置条件

必须在 macOS + Xcode 环境执行。

```bash
xcodebuild -version
```

首次使用需要给脚本执行权限：

```bash
chmod +x scripts/package_ios.sh
```

## 模拟器构建

适合验证代码是否能编译：

```bash
./scripts/package_ios.sh simulator
```

## Archive

需要真实 iOS 签名环境：

```bash
TEAM_ID=你的Apple开发者TeamID ./scripts/package_ios.sh archive
```

## 导出 IPA

开发包：

```bash
TEAM_ID=你的Apple开发者TeamID ./scripts/package_ios.sh ipa development
```

App Store 包：

```bash
TEAM_ID=你的Apple开发者TeamID ./scripts/package_ios.sh ipa app-store
```

Ad Hoc 包：

```bash
TEAM_ID=你的Apple开发者TeamID ./scripts/package_ios.sh ipa ad-hoc
```

## 产物目录

每次执行会生成类似目录：

```text
versions/v1.0_20260527_0030/
├── RELEASE_NOTES.md
├── logs/
│   ├── clean.log
│   ├── simulator_build.log
│   ├── archive.log
│   └── export_ipa.log
└── ipa/
    └── amz_profit_calculator.ipa
```

Archive 默认位置：

```text
build/amz_profit_calculator.xcarchive
```

## 可覆盖参数

```bash
SCHEME=amz_profit_calculator
PROJECT=amz_profit_calculator.xcodeproj
CONFIGURATION=Release
TEAM_ID=你的Apple开发者TeamID
BUNDLE_ID=com.vvitem.amzprofitcalculator
EXPORT_METHOD=development|app-store|ad-hoc|enterprise
```

示例：

```bash
TEAM_ID=ABCDE12345 EXPORT_METHOD=app-store ./scripts/package_ios.sh ipa
```

## 常见问题

### 1. Archive 签名失败

检查 Xcode 中：

```text
Signing & Capabilities → Team
Bundle Identifier
Provisioning Profile
```

### 2. 找不到 Scheme

项目已提交共享 Scheme：

```text
amz_profit_calculator.xcodeproj/xcshareddata/xcschemes/amz_profit_calculator.xcscheme
```

如果本地仍找不到，执行：

```bash
xcodebuild -list -project amz_profit_calculator.xcodeproj
```

### 3. App Store 导出失败

需要 Apple Developer Program 账号、正确证书和 App Store provisioning profile。

### 4. 只想验证代码

使用模拟器构建即可，不需要签名：

```bash
./scripts/package_ios.sh simulator
```
